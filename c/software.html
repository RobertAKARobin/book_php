People think you need to buy a lot of fancy software to make websites. You don't. You only need two pieces of software, both of which are free and already on your computer: a web browser — Chrome, Firefox, Internet Explorer, or Safari — and a plain text editor — Notepad on Windows and TextEdit on Macs.

When you look at something in a plain text editor, you're seeing everything in that file. If you make a new document in Microsoft Word, leave it totally blank, save it as a .doc or .docx file, and open that file in a plain text editor, you'll see a huge page of gobbledygook. That's because a blank Word document isn't actually blank.

Don't write webpages in Microsoft Word. If you open a Word file in TextEdit or Notepad you'll see a lot of gobbledygook. That's because Word files contain lots of hidden code that tells the Word program how to display the file on your screen. Plain text editors show you everything that's in a file, hidden code included.

A blank Microsoft Word document isn't really blank: it contains lots of hidden code that helps the Word program figure out how to display the document. When you write something in plain text what you see on your screen is everything that's in the file. That's why a blank Word file takes up 29 kilobytes of memory and a blank .txt file takes up 0 bytes. That's also why if you open a Word .docx file in Notepad or TextEdit you'll see a huge page of gobbledygook: you're seeing the hidden code.

Notepad or TextEdit might be set to work in rich text. Rich text files are like Word files in that they have hidden code, and they usually end in .rtf. To switch to plain text you'll need to look in the settings or preferences for a plain text format option. Usually plain text editors show everything in a monospace font (which means all the letters have the same width) and don't use any bolding or different font sizes.

There are lots of other plain text editors out there, many of them free. A lot of programmers use text editors that include a feature called syntax highlighting. This turns different bits of code into different colors so you can tell them apart easily. These colors aren't saved in the files themselves, so the files are still all plain text.

Webpage files end in .html, but otherwise they're just like plain text files. index.html should probably be the first page you make. Just create a new document in your text editor and "save" it as index.html.

HTML divides content into semantic elements (chunks). Each element begins and ends with a tag, like <div>blah blah</div>. div elements are the most frequently-used HTML element, with "div" just standing for "division," as in a division of content.

Let's say I have this text:

Hamlet's Soliloquoy, by Bill Shakespeare. To be, or not to be: that is the question.

That pretty clearly contains two chunks with different meanings: there's a header — something that introduces a section of content — and then the main content.

<header>Hamlet's Soliloquoy, by Bill Shakespeare.</header>

<main>To be, or not to be: that is the question.</main>

HTML's "grammar" is simple:

Every element needs to be closed.
It needs a beginning tag and an end tag. Otherwise your web browser doesn't know where an element ends and you get stuff like this.
Elements have to nest.
...just like directories and files.
This is wrong: <div> <strong> </div> </strong>.
This is right: <div> <strong> </strong> </div>.

Beyond that, HTML is just figuring out what tags to use when.

You might have noticed one goofy tag in there: <img />. That's called a self-closing tag and there are only three or four of them. They indicate elements that can't have any text content and therefore don't need both a start and end tag to show the boundaries of the content.

<img>For example, this</img> would tell your web browser this bit of text is an image, and that doesn't make any sense.

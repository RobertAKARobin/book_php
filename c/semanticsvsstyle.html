There are several elements besides <div>, each with a unique semantic value. For instance, <strong> indicates important text that would be read in a strong voice. <em> indicates text that would be read with emphasis. <div> is what you use when none of the other tags seems to fit.

The biggest rule in web design is that semantics and style should be separate. You should use HTML just to define the semantic value of stuff, and CSS just to define the stylistic value.

A couple reasons HTML tags are based on semantics, rather than offering tags like <bold> or <underline> or <orange>:

Form follows function
The style of an element almost always depends on its semantic value. It's much more likely that you'd want every header to be underlined than everything that's underlined to be a header.
Learnability
Let's say <orange> is a tag. What if you want to use a few different shades of orange? What tags would you use? We'd need to have a tag for every color, which would result in HTML having about a gazillion tags.
Accessibility

Defining semantics is even more important when people don't understand your words. If someone who doesn't speak any English saw Hamlet in English, the only way they'll be able to figure out what anyone's saying is by listening for semantic cues: Is something being yelled? Sobbed? Whispered?

If a blind person is reading your site in Braille — which happens more often than you'd think — an <orange> tag wouldn't be very useful. A <strong> tag would be much better at helping them figure out what you're trying to communicate.

Manageability
Stylesheets let you change the styles for all of one type of element in one line of code. They also make it easy to try different themes on a page. For instance, if you're not sure whether you want your webpage's color scheme to be dark or light you can make one stylesheet for each and just swap them out — no need to touch your HTML.
Scalability
We'll see in a bit how to easily use the same stylesheet on a bunch of pages without having to do lots of copying and pasting. This means you can easily style an entire collection of webpages the exact same way.

CSS didn't come along until HTML had been around for a while. Before then HTML had lots of semantic tags, like <b> for 'bold' and <i> for 'italic'. You can still see that on some sites, like the one for the 1996 movie Space Jam: www2.warnerbros.com/spacejam/movie/jam.htm

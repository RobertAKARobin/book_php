If I buy RobinIsACoolDude.com and immediately open that up in a web browser I'll get a "page not found" error. A domain without a server is like a phone number without a phone. A server without a host computer is like a phone without someone to answer it.

Host really just means a computer that's connected to the Internet. But when you're talking about web design host means the computer storing your website's files and running its server. Your server is the computer program that "lives" on your host and answers requests for the files inside it. Remember the secretary with the phone and the filing cabinet?

You can turn pretty much any computer into a host. You can even run a server on your own personal computer, which is called self-hosting. I don't know anyone who actually does that, because if you're self-hosting and turn off your computer, your whole website goes down. If you're using your computer to watch a Lord of the Rings marathon it will serve requests really slowly.

There are lots of different server programs — Apache, Oracle, and so on. They can be really hard to install and keep running, and how exactly they work is pretty complicated. Fortunately, like a lot of things in web design, you don't need to know how they work in order to work with them. Until you get into really advanced web design, what I've talked about so far is all you need to know.

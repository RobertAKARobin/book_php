If I buy the domain RobinIsACoolDude.com and immediately open that up in a web browser I'll get a "page not found" error. A domain without a server is like a phone number without a person to answer it. A server without a host computer is like a secretary without a phone.

Host really just means a computer that's connected to the Internet. But when you're talking about web design a host is the computer storing your website's files and running its server. Your server is the computer program that "lives" on your host and answers requests for the files inside it.

To recap: The domain is the phone number. The server is the secretary. The host is the secretary's office, containing a phone and a filing cabinet. The URL is the number you're calling and what you're saying to the secretary. Every time you call is a new request.

You can run a server on pretty much any computer, even your own personal laptop, which is called self-hosting. I don't know anyone who actually does that, because if you're self-hosting and turn off your computer, your whole website goes down. If you're using your computer to watch a Lord of the Rings marathon it will serve requests really slowly.

There are lots of different server programs — Apache, Oracle, and so on. They can be really hard to install and keep running, and how exactly they work is pretty complicated.

Fortunately, like a lot of things in web design, you don't need to know how they work in order to work with them. What I've talked about so far is all you need to know until you get into really advanced web design.

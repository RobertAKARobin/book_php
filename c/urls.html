www.wsmith1.com/raos/freshprince.html is a URL, which is the fancy name for "web address."

8675309.com is a domain. It's like a website's phone number. www is a subdomain. It's like a phone number's extension (as in, Dial '4' for Customer Support). Some sites have lots of subdomains — Craigslist has one for each region it serves, like newyork.craigslist.com — but most just stick with www.

The domain and subdomain together are the base URL. After that comes the path — the directions to the file you want. Anything followed by a slash (/) is a directory, which is the fancy name for a folder. The path usually ends in the name of the file you want.

So when I type www.8675309.com/raps/freshprince.html into my web browser, it's like I'm calling a secretary at 867-5309 and saying, I want you to open your filing cabinet, look for a folder called raps, and look in there for a file called freshprince.html.

You can name a file or directory whatever you want alphanumeric characters (a-z, A-Z, and 0-9), undercores (_), and hyphens (-). The .html at the end of freshprince.html is a file extension, which says what type of file it is. Every type of file has its own special extension.

People like to say backslash when they're reading URLs. That drives me crazy. A slash is / and a backslash is \. They do completely different things in the computer world, and you'll almost never find a backslash in a URL.

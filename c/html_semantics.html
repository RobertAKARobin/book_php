A webpage's code is a bunch of text broken up into chunks, based on the purpose or semantic value of each chunk, the way a joke can be chunked into a set-up and a punchline.

You do this "chunking" with a kind of code called HTML, which stands for Hypertext Markup Language.

When my teacher marks up my paper, it means he's writing comments on my paper: This should be a new paragraph.... This is incorrect.... This is really good.... He isn't changing what I wrote. He's taking a chunk of what I wrote and making a suggestion about its semantic value.

The script for a movie or a play is a bunch of text broken up into lines with instructions for who says which line. Those lines might be broken up into smaller chunks with instructions for how each chunk should be said, like angrily or to the audience.

Computers make webpages the way a director makes a movie from a script. You give the computer a bunch of text broken up into chunks. Each chunk has instructions for what the computer should do with that chunk. The computer follows the instructions to make different chunks bigger, smaller, flashy colors, put graphics in them... and then shows the result on your screen.

HTML is a way of marking up text so that a computer can understand that different parts of your webpage's text have different purposes.

<div class="paragraph">An HTML chunk is called an <div class="define">element</div>. It begins and ends with <div class="define">tags</div> — a start tag and an end tag.</div>

Elements have two rules:

  1. They have to be closed. Every start tag gets an end tag.
  2. They have to nest, like directories and files. An element can't start inside another element and end outside it.
